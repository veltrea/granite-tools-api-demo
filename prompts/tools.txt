### open
Read a file and display it in a scrollable editor window (100 lines at a time).
Output format: line numbers (`N | content`), total line count, and visible range.
You MUST open a file before you can edit, insert, scroll, or search within it.
- path (string, required): Relative path from workspace root.
- line_number (integer, optional): Center the view on this line.
Example: [TOOL_CALLS]open: {"path": "src/main.rs"}[/TOOL_CALLS]
Example: [TOOL_CALLS]open: {"path": "src/main.rs", "line_number": 50}[/TOOL_CALLS]

### goto
Move the editor window to show the specified line number. A file must already be open.
- line_number (integer, required): The 1-based line number to jump to.
Example: [TOOL_CALLS]goto: {"line_number": 100}[/TOOL_CALLS]

### scroll_up / scroll_down
Scroll the editor window by one page (100 lines). A file must already be open.
No parameters.
Example: [TOOL_CALLS]scroll_down: {}[/TOOL_CALLS]

### create
Create a new empty file and open it in the editor. Parent directories are created automatically.
- filename (string, required): Relative path for the new file.
Example: [TOOL_CALLS]create: {"filename": "utils/helpers.py"}[/TOOL_CALLS]

### edit
Replace text in the currently open file. You MUST open the file first.
Replaces the first occurrence of `search` with `replace`. Set `replace-all` to true for all occurrences.
The `search` string must exactly match existing content (including whitespace and indentation).
Returns an error if `search` is not found — double-check with `open` or `search_file` first.
- search (string, required): The exact text to find in the current file.
- replace (string, required): The replacement text.
- replace-all (boolean, optional): Replace all occurrences instead of just the first. Default: false.
Example: [TOOL_CALLS]edit: {"search": "Hello world", "replace": "Hello, World!"}[/TOOL_CALLS]

### insert
Insert text into the currently open file. Appends at the end by default, or after the specified line.
- text (string, required): The text to insert (can be multiple lines with \n).
- line (integer, optional): Insert after this line number (1-based). Omit to append at end.
Example: [TOOL_CALLS]insert: {"text": "# New section\ndef helper():\n    pass", "line": 10}[/TOOL_CALLS]

### find_file
Search for files by name pattern using glob syntax (*, ?). Returns up to 50 matching file paths.
- file_name (string, required): Glob pattern to match against file names (e.g., "*.rs", "config.*").
- dir (string, optional): Subdirectory to search in. Default: workspace root.
Example: [TOOL_CALLS]find_file: {"file_name": "*.json"}[/TOOL_CALLS]

### search_dir
Search for a text string across all files in a directory. Returns matching lines with file paths and line numbers. Up to 50 results.
- search_term (string, required): The text to search for (exact substring match).
- dir (string, optional): Subdirectory to search in. Default: workspace root.
Example: [TOOL_CALLS]search_dir: {"search_term": "TODO", "dir": "src"}[/TOOL_CALLS]

### search_file
Search for a text string within a specific file or the currently open file. Returns matching lines with line numbers.
- search_term (string, required): The text to search for.
- file (string, optional): File to search. Default: currently open file.
Example: [TOOL_CALLS]search_file: {"search_term": "fn main", "file": "src/main.rs"}[/TOOL_CALLS]

### filemap
Show the structure of a source file: function/class/method signatures with line numbers, bodies collapsed. Useful for understanding a file without reading every line.
Works with Python, Rust, JavaScript, TypeScript, Go, and Java.
- file_path (string, required): Relative path to the file.
Example: [TOOL_CALLS]filemap: {"file_path": "src/tools.rs"}[/TOOL_CALLS]

### submit
Mark the current task as complete. Call this when you have finished all requested changes.
No parameters.
Example: [TOOL_CALLS]submit: {}[/TOOL_CALLS]

### allow_outside_workspace
Enable or disable access to files outside the workspace directory.
By default, all file operations are restricted to the workspace root. Call this tool with allow=true if the user asks you to access absolute paths or files outside the workspace.
- allow (boolean, optional): true to enable, false to disable. Default: true.
Example: [TOOL_CALLS]allow_outside_workspace: {"allow": true}[/TOOL_CALLS]
